vue/重载页面

2020年6月28日 · 阅读需 3 分钟

单页应用重新加载页面

1.使用Vue Router的$router.go(0)方法:

你可以在组件内通过调用Vue实例的this.$router.go(0)来刷新当前页面。这会将当前路由的组件重新加载，模拟页面刷新的效果。

methods: {
  reloadPage() {
    this.$router.go(0);
  }
}

2.使用window.location.reload()方法: 如果不介意完全重新加载整个页面（包括JavaScript、CSS等静态资源），可以使用浏览器的原生方法window.location.reload()来实现刷新。

methods: {
    reloadPage() {
        window.location.reload();
    }
}

3.Vue的Composition API中的状态重置:

在使用Vue 3的Composition API时，可以通过重置组件的响应式状态来实现类似于刷新的效果。这通常涉及到将状态重置为其初始值，但不会重新加载整个页面。

4.使用provide和inject进行组件刷新:

通过Vue的依赖注入系统，可以在父组件中提供一个刷新方法，并在需要刷新的子组件中注入这个方法，从而实现刷新逻辑的传递。

// 父组件
export default {
  provide() {
    return {
      reload: this.reload
    };
  },
  methods: {
    reload() {
      // 重置状态逻辑
    }
  }
};

// 子组件
export default {
  inject: ['reload'],
  methods: {
    refreshPage() {
      this.reload();
    }
  }
};

5.利用Vue的热重载功能:

在开发环境中，Vue CLI和Vue Loader支持热模块替换（HMR, Hot Module Replacement），这会在你编辑代码后自动更新页面，而无需手动刷新。但在生产环境中，热重载通常不可用，这时应采用上述方法之一。

结: 根据应用场景选择合适的刷新策略，如果是为了响应用户操作或状态变更，使用Vue Router的$router.go(0)或状态重置可能是更好的选择。如果是希望模拟浏览器刷新行为或解决特定问题，则window.location.reload()更为直接。

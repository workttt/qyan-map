响应式​

ref()               其实就是包装成一层 { value: 创建的变量 } 对象的结构, 但是具有深层响应性, 所有字段都是响应式的
shallowRef()        shallow ref 来放弃深层响应性
reactive()          转换为响应式代理, 返回的是一个原始对象的 Proxy，它和原始对象是不相等的
shallowReactive()   // API 可以选择退出深层响应性。

reactive() 的局限性: 有限的值类型, 不能替换整个对象, 对解构操作不友好
ref                 解包, ref对象赋值给其他变量的实际情况

注: ref对象的引用问题归根结底就是js引用类型的地址问题

计算属性​

= computed(()=> return...)
import { computed } from 'vue'

const author = reactive({
  name: 'John Doe',
  books: [
    'Vue 2 - Advanced Guide',
    'Vue 3 - Basic Guide',
    'Vue 4 - The Mystery'
  ]
})

// 一个计算属性 ref
const publishedBooksMessage = computed(() => {
  return author.books.length > 0 ? 'Yes' : 'No'
})

Vue 的计算属性会自动追踪响应式依赖。

它会检测到 publishedBooksMessage 依赖于 author.books，所以当 author.books 改变时，任何依赖于 publishedBooksMessage 的绑定都会同时更新。

计算属性值会基于其响应式依赖被缓存。一个计算属性仅会在其响应式依赖更新时才重新计算。

这意味着只要 author.books 不改变，无论多少次访问 publishedBooksMessage 都会立即返回先前的计算结果，而不用重复执行 getter 函数。

可写计算属性​

const fullName = computed({
  // getter
  get() {
    return firstName.value + ' ' + lastName.value
  },
  // setter
  set(newValue) {
    // 注意：我们这里使用的是解构赋值语法
    [firstName.value, lastName.value] = newValue.split(' ')
  }
})

不要改变其他状态、在 getter 中做异步请求或者更改 DOM！

  • Vue3 提供了 Composition API，这是一组基于函数的 API，允许灵活的组合组件逻辑。
  • Vue3 的生命周期钩子更加细粒度，提供了更多的控制可能性。

侦听器 *​

可以直接侦听一个响应式对象 ref, reactive

目的：专门监视指定的响应式源（ref、reactive 对象或getter函数）的变化。
用法：你需要传递你想要监视的源和一个回调函数，当源发生变化时执行该回调函数。回调函数接收新值和旧值。
控制：提供更多控制权，因为你可以精确定义触发副作用的内容。

watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.includes('?')) {
    loading.value = true
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    } finally {
      loading.value = false
    }
  }
})
const x = ref(0)
const y = ref(0)

// 单个 ref
watch(x, (newX) => {
  console.log(`x is ${newX}`)
})

// getter 函数
watch(
  () => x.value + y.value,
  (sum) => {
    console.log(`sum of x + y is: ${sum}`)
  }
)

// 多个来源组成的数组
watch([x, () => y.value], ([newX, newY]) => {
  console.log(`x is ${newX} and y is ${newY}`)
})

不能直接侦听响应式对象的属性值 watchEffect() 目的：自动跟踪其函数体中遇到的所有响应式依赖，并在任何依赖发生变化时运行该函数。 用法：你只需要定义一个函数，Vue 会处理其余的，跟踪在此函数中使用的任何响应式属性。 控制：对触发效果的控制较少，因为它会自动跟踪所有依赖。这对于应基于多个源更新的效果或当事先不知道确切依赖时非常有用。

  1. 清理副作用 在 watchEffect 的回调函数中，可以进行一些需要清理的副作用操作，比如设置定时器、订阅事件等。Vue 提供了一个 onInvalidate 函数用于注册清理函数。
watchEffect((onInvalidate) => {
  const timer = setTimeout(() => {
    console.log(`count is ${count.value}`);
  }, 1000);

  // 清理函数
  onInvalidate(() => {
    clearTimeout(timer);
  });
});
  1. 利用 flush 选项控制副作用的触发时机 watchEffect 接受一个配置对象，其中 flush 属性可以控制副作用的触发时机。默认为 'pre'（同步触发），还可以设置为 'post'（在 DOM 更新之后触发）或 'sync'（与响应式效果同步触发）。
watchEffect(() => {
  console.log(`count is ${count.value}`);
}, { flush: 'post' });
  1. 利用 onTrack 和 onTrigger 进行调试 watchEffect 的配置对象还可以接受 onTrack 和 onTrigger 两个函数，这两个钩子用于调试，可以帮助你了解响应式系统的行为。
watchEffect(() => {
  console.log(`count is ${count.value}`);
}, {
  onTrack(e) {
    console.log('Tracking', e);
  },
  onTrigger(e) {
    console.log('Trigger', e);
  }
});

组件基础​

defineProps(['title'])
defineProps({
  propA: Number,
  propB: [String, Number],
  propC: { type: Number, required: true, default: 100 },
  // 对象类型的默认值
  propE: {
    type: Object,
    // 对象或数组的默认值 // 必须从一个工厂函数返回。 // 该函数接收组件所接收到的原始 prop 作为参数。
    default(rawProps) {
      return { message: 'hello' }
    }
  },
  // 自定义类型校验函数
  // 在 3.4+ 中完整的 props 作为第二个参数传入
  propF: {
    validator(value, props) {
      // The value must match one of these strings
      return ['success', 'warning', 'danger'].includes(value)
    }
  },
  // 函数类型的默认值
  propG: {
    type: Function,
    // 不像对象或数组的默认，这不是一个 // 工厂函数。这会是一个用来作为默认值的函数
    default() {
      return 'Default function'
    }
  }
})

动态组件​

!!!当对象或数组作为 props 被传入时，虽然子组件无法更改 props 绑定，但仍然可以更改对象或数组内部的值。这是因为 JavaScript 的对象和数组是按引用传递，而对 Vue 来说，禁止这样的改动，虽然可能生效，但有很大的性能损耗，比较得不偿失。

这种更改的主要缺陷是它允许了子组件以某种不明显的方式影响父组件的状态，可能会使数据流在将来变得更难以理解。在最佳实践中，你应该尽可能避免这样的更改，除非父子组件在设计上本来就需要紧密耦合。在大多数场景下，子组件应该抛出一个事件来通知父组件做出改变。

基础类型​

  1. any 任意类型

声明为 any 的变量可以赋予任意类型的值。

  1. unknown

unknown 类型表示任何值。

与 any 类型类似，但更安全，因为对 unknown 类型的值进行操作之前，需要进行类型检查。

  1. number 数字类型

双精度 64 位浮点值。它可以用来表示整数和分数。

let binaryLiteral: number = 0b1010; // 二进制
let octalLiteral: number = 0o744;    // 八进制
let decLiteral: number = 6;    // 十进制
let hexLiteral: number = 0xf00d;    // 十六进制
  1. string 字符串类型

一个字符系列，使用单引号（'）或双引号（"）来表示字符串类型。反引号（`）来定义多行文本和内嵌表达式。

let name: string = "你的名字";
let years: number = Date.now().getFullYear();
let words: string = `您好，${ name }, 今年是${ years } `;
  1. boolean 布尔类型

表示逻辑值：true 和 false。

let flag: boolean = true;
  1. 数组

数组类型由数组元素类型和方括号组成。

// 在元素类型后面加上[]
let arr1: number[] = [1, 2];

// 或者使用数组泛型
let arr2: Array<number> = [1, 2];

// readonly: 不允许更改，只可以从中读取而不能修改
const codeNames: readonly string[] = ['Coding', 'God']
  1. 元组

元组类型用来表示已知元素数量和类型的数组，各元素的类型不必相同，对应位置的类型需要相同。

let x: [string, number];
x = ['Runoob', 1];    // 运行正常
x = [1, 'Runoob'];    // 报错
console.log(x[0]);    // 输出 Runoob

需要注意的是，当对元组使用某些方法（如 push, pop, shift, unshift）改变其长度或类型时，可能会违反原始元组的类型定义，

因为这些操作会改变元组的结构，从而可能导致类型错误。

在实际编程中，应谨慎使用这类会修改元组结构的方法，以保持类型安全。

  1. enum 枚举

枚举类型用于定义数值集合。

enum Color {Red, Green, Blue};
let c: Color = Color.Blue;
console.log(c);    // 输出 2
  1. void

用于标识方法返回值的类型，表示该方法没有返回值。

function hello(): void {
    alert("Hello");
}
  1. null

表示对象值缺失。 在 JavaScript 中 null 表示 "什么都没有"。

null是一个只有一个值的特殊类型。表示一个空对象引用。

用 typeof 检测 null 返回是 object。

  1. undefined

用于初始化变量为一个未定义的值。

typeof 一个没有值的变量会返回 undefined。

Null 和 Undefined 是其他任何类型（包括 void）的子类型，可以赋值给其它类型。

而在TypeScript中启用严格的空校验（--strictNullChecks）特性，就可以使得null 和 undefined 只能被赋值给 void 或本身对应的类型

// 启用 --strictNullChecks
let x: number;
x = 1; // 编译正确
x = undefined;    // 编译错误
x = null;    // 编译错误

// 正确的声明
let x2: number | null | undefined;
x2 = 1; // 编译正确
x2 = undefined;    // 编译正确
x2 = null;    // 编译正确
  1. never

never 是其它类型（包括 null 和 undefined）的子类型，代表从不会出现的值。

这意味着声明为 never 类型的变量只能被 never 类型所赋值，在函数中它通常表现为抛出异常或无法执行到终止点（例如无限循环）

let x: never;
let y: number;

// 编译错误，数字类型不能转为 never 类型
x = 123;

// 运行正确，never 类型可以赋值给 never类型
x = (()=>{ throw new Error('exception')})();

// 运行正确，never 类型可以赋值给 数字类型
y = (()=>{ throw new Error('exception')})();

// 返回值为 never 的函数可以是抛出异常的情况
function error(message: string): never {
    throw new Error(message);
}

// 返回值为 never 的函数可以是无法被执行到的终止点的情况
function loop(): never {
    while (true) {}
}

注意：TypeScript 和 JavaScript 没有整数类型。​

函数重载​

重载是方法名字相同，而参数不同，返回类型可以相同也可以不同。

每个重载的方法（或者构造函数）都必须有一个独一无二的参数类型列表。

  • 参数类型不同
  • 参数数量不同
  • 参数顺序不同

语法：Type1|Type2|Type3

console.log("字符串为 " + val)

// 参数
function x(name: string|string[]) {

接口​

  • 推荐I开头命名
  • 用于指定对象类型
  • ?: 表示可选
  • readonly 只读修饰
interface IPerson {
  // 只读属性
  readonly id: string,
  firstName: string, 
  lastName: string, 
  age?: number,
  sayHi: ()=>string 
} 
 
var customer:IPerson = { 
    firstName:"Tom",
    lastName:"Hanks", 
    sayHi: ():string =>{return "Hi there"} 
} 
 
console.log("Customer 对象 ") 
console.log(customer.firstName) 
console.log(customer.lastName) 
console.log(customer.sayHi())
联合类型和接口
interface RunOptions { 
    program:string; 
    commandline:string[]|string|(()=>string); 
} 
 
// commandline 是字符串
var options:RunOptions = {program:"test1",commandline:"Hello"}; 
console.log(options.commandline)  
 
// commandline 是字符串数组
options = {program:"test1",commandline:["Hello","World"]}; 
console.log(options.commandline[0]); 
console.log(options.commandline[1]);  
 
// commandline 是一个函数表达式
options = {program:"test1",commandline:()=>{return "**Hello World**";}}; 
 
var fn:any = options.commandline; 
console.log(fn());

接口的继承​

Child_interface_name extends super_interface_name

Child_interface_name extends super_interface1_name, super_interface2_name,…,super_interfaceN_name

interface Person {
    age:number
}

interface Musician extends Person {
    instrument:string
}

var drummer = <Musician>{};
drummer.age = 27
drummer.instrument = "Drums"
console.log("年龄:  "+drummer.age)
console.log("喜欢的乐器:  "+drummer.instrument)

泛型​

动态类型

泛型（Generics）是一种强大的特性，允许你在定义函数、接口或类时将类型作为参数，从而创建可重用的组件，这些组件可以在不同的上下文中使用多种类型。

泛型的主要目的是提高代码的灵活性和类型安全性。

  1. 泛型函数可以在调用时指定具体类型：
泛型函数
function identity<T>(arg: T): T {
    return arg;
}
let myIdentity: <T>(arg: T) => T = identity;
let output = identity<string>("hello"); // T 被替换为 string 类型

identity(100)
identity('你好~')
identity(false)
  1. 泛型类允许类的实例化时指定类型参数：
泛型类
class GenericClass<T> {
  private value: T;
  constructor(val: T) {
    this.value = val;
  }
  getValue(): T {
    return this.value;
  }
}
let genericNumber = new GenericClass<number>(5);
console.log(genericNumber.getValue()); // 输出数字 5
  1. 接口也可以是泛型的，为实现它的类或对象提供类型约束：
泛型接口
interface GenericInterface<T> {
    value: T;
}
class ImplementsGeneric implements GenericInterface<string> {
    value: string;
    constructor(val: string) {
        this.value = val;
    }
}

泛型约束​

有时你可能需要对泛型参数施加一些限制，确保它满足特定的条件，比如实现了某个接口。

interface IWithLength {
  length: number;
}

function returnLength<T extends IWithLength>(item: T): void {
  console.log(item.length);
}

var b: boolean = false
returnLength([1, 2, 3])
returnLength(b) // 提示错误

在这个例子中，T 必须是一个具有 length 属性的类型。
